## 为什么会出现“接口401国内”的错误？

先理解一下报错原理，不然解决了一个 401，下回换个程序还是报错。

当你在本地用代码调用一个海外 API 的时候，流程是这样的：

1.  你的程序发出 HTTP 请求。
2.  请求要跨过大陆的网络边界（俗称“翻墙”）。
3.  请求到达海外服务器收到请求，验证你的 API Key。
4.  如果验证通过，返回 200；如果没通过，返回 401。

问题出在第 2 步。国内个人或者小团队的“翻墙”网关并不稳定。网关不稳定、IP 被限流、请求中途被阻断，都会导致请求发送不完整或者被篡改。这时候，即便你手里的 Key 真金白银充了几百美刀，服务器端收到的依然是一个“未授权”的请求”或“非法请求”，自然给你甩一个 401。

所以，“接口401国内”的根源，**从来不是你 Key 的问题，而是你连接海外服务器的那条路断了。** 不打这个结，换一万次重试也白搭。
