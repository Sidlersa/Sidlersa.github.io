## 完整可运行代码（Maven 项目）

我假设你已经有一个能跑起来的 Java 8+ 项目了。如果没有，新建一个 Maven 项目，然后跟着来。

### 1. 引入 Maven 依赖

首先，在你的 `pom.xml` 里加入 openai 的 Java 客户端依赖。虽然官方推荐用 `com.theokanning.openai-gpt3-java` 这个库，但我用过之后发现它经常有小 bug。我比较推荐用 **OpenAI 官方最新的 Java SDK**，或者如果你不愿意折腾，直接用 OkHttp 自己去封装。

### 2. 核心调用代码：`Glm4JavaDemo.java`

把下面的代码拷进你的 Java 文件里，只需要改两处地方：
- **替换 `YOUR_API_KEY`**：改成你在[千聚ai中转站](https://www.qianjuai.com/)拿到的那个 Key。
- **替换 `YOUR_ACCOUNT_URL`**：千聚的 API 接口连接就是 `https://www.qianjuai.com/v1`。

java
import okhttp3.*;
import com.google.gson.Gson;
import com.google.gson.JsonObject;

import java.io.IOException;
import java.util.concurrent.TimeUnit;

public class Glm4JavaDemo {

    // ！核心配置：务必替换为你自己的 API Key 和 Base URL
    private static final String BASE_URL = "https://www.qianjuai.com/v1";
    private static final String API_KEY = "你的[千聚api中转站](https://www.qianjuai.com/)Key写在这里";
    private static final String MODEL_NAME = "glm-4.5";

    public static void main(String[] args) {
        OkHttpClient client = new OkHttpClient.Builder()
                .connectTimeout(60, TimeUnit.SECONDS) // 模型推理时间长，超时设置长一点
                .readTimeout(120, TimeUnit.SECONDS)
                .build();
        Gson gson = new Gson();

        // 构建请求体，完全兼容 OpenAI Chat Completion 格式
        JsonObject requestBody = new JsonObject();
        requestBody.addProperty("model", MODEL_NAME);

        // 添加消息
        JsonObject message = new JsonObject();
        message.addProperty("role", "user");
        message.addProperty("content", "请用 50 字以内解释为什么 Java 开发人员喜欢用 API 中转站。");
        // messages 数组
        requestBody.add("messages", gson.toJsonTree(new JsonObject[]{message}));

        // 设置可选参数
        requestBody.addProperty("max_tokens", 500);
        requestBody.addProperty("temperature", 0.7);

        // 构建请求
        Request request = new Request.Builder()
                .url(BASE_URL + "/chat/completions")
                .addHeader("Authorization", "Bearer " + API_KEY)
                .addHeader("Content-Type", "application/json")
                .post(RequestBody.create(
                        MediaType.parse("application/json; charset=utf-8"),
                        gson.toJson(requestBody)))
                .build();

        try (Response response = client.newCall(request).execute()) {
            if (!response.isSuccessful()) {
                System.err.println("请求失败，HTTP Code: " + response.code());
                System.err.println("错误信息: " + response.body().string());
                return;
            }
            String responseBody = response.body().string();
            System.out.println("GLM-4.5 返回的原始 JSON: ");
            System.out.println(responseBody);

            // 解析并提取回复内容
            JsonObject jsonResponse = gson.fromJson(responseBody, JsonObject.class);
            String replyContent = jsonResponse
                    .getAsJsonArray("choices")
                    .get(0).getAsJsonObject()
                    .getAsJsonObject("message")
                    .get("content")
                    .getAsString();
            System.out.println("\n最终回复内容: ");
            System.out.println(replyContent);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

**① Base URL 的尾部必须有 `/v1`**
[千聚ai中转站](https://www.qianjuai.com/)的 API 接口链接是 `https://www.qianjuai.com/v1`。如果你的代码里写的是 `http://www.qianjuai.com` 少了 `/v1`，请求会失败。因为在标准 OpenAI 格式中，路由 `/chat/completions` 是挂在 `/v1` 路径下的。
