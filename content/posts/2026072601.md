## 为什么你的 API 密钥容易泄露？先看看这些常见坑

很多新手开发者（甚至一些资深开发者）犯的错误，其实就那几类。

**最常见的是直接把密钥写死在代码里**。比如在 `index.js` 里直接写 `const apiKey = "sk-xxxxxxxxxxxx";`，然后分享代码、上传到 GitHub，或者在调试时截图发到群里。如果你的仓库是公开的，别人通过爬虫扫描代码库里的 API Key 非常容易，几分钟就能发现并滥用，导致你的账号被运营商判定为异常使用而直接封禁。

**其次是把密钥写错地方**。有人会在 `.env` 文件里放密钥，但忘了把 `.env` 添加到 `.gitignore`，结果同一次 `git push` 把密钥也公之于众。还有人会把密钥存在环境变量文件里，但文件权限没设好，被服务器上的其他进程读取。

> 我见过最离谱的一次：一个开发者把 API Key 写在 README.md 的示例代码里，标题还写着“openai node.js demo”——这就相当于主动把钥匙挂在门口。

## 正确的做法：环境变量 + .env 文件 + .gitignore 配置

想要安全调用 OpenAI 接口，第一步就是学会隔离密钥。核心原则很简单：**代码里永远不出现密钥，密钥只存放在环境变量中**。

具体怎么做呢？一个标准的主流做法是这样的：

1. **在你的项目根目录下创建 `.env` 文件**（注意文件前面有个点，不是 `.env.example`）
2. **在 `.env` 文件中写入密钥变量**（这个文件不会提交到 git）
3. **创建 `.env.example` 文件**（只写变量名，不填值），让协作者知道需要配置哪些密钥
4. **把 `.env` 加入 `.gitignore`**，确保它永远不会被提交到版本控制中
5. **用 `process.env` 来读取密钥**

这个流程几乎被所有主流 Node.js 项目采用。用好了，你才能放心地写代码、发 Pull Request、甚至公开分享你的仓库，而不用担心密钥泄露。

### 第一步：安装依赖

- **openai**：调用 OpenAI 兼容接口的官方 SDK
- **dotenv**：读取 `.env` 文件里的变量注入到 `process.env`

### 第二步：创建 `.env` 和 `.env.example`

**`.env` 文件（真正的密钥放这里，绝对不要提交到 Git）：**

**`.env.example` 文件（用于告诉合作者：需要配两个变量）：**

### 第三步：配置 `.gitignore`

在项目根目录找到或创建 `.gitignore`，加上一行：

gitignore
.env


这样当你 `git add .` 时，`.env` 文件会自动被 Git 忽略。

### 第四步：编写安全的调用代码

在 `index.js` 中写如下代码：

javascript
// index.js
import OpenAI from "openai";
import dotenv from "dotenv";

// 加载.env文件（必须在调用OpenAI之前）
dotenv.config();

const OPENAI_API_KEY = process.env.OPENAI_API_KEY;
const BASE_URL = process.env.BASE_URL;

if (!OPENAI_API_KEY) {
  throw new Error("[错误] 未检测到API密钥，请检查.env文件中的OPENAI_API_KEY是否已设置。");
}
if (!BASE_URL) {
  throw new Error("[错误] 未检测到BASE_URL，请检查.env文件。");
}

// 使用千聚中转站API地址
const openai = new OpenAI({
  apiKey: OPENAI_API_KEY,
  baseURL: BASE_URL,
});

async function main() {
  try {
    const completion = await openai.chat.completions.create({
      model: "gpt-4o", // 你可以在千聚支持列表中找到其他500+模型的标识
      messages: [
        { role: "system", content: "你是一个安全的AI助手，请用中文回答。" },
        { role: "user", content: "Hello ChatGPT, 请用一句话介绍一下如何安全地管理API密钥。" },
      ],
      max_tokens: 200,
    });

    console.log("=== AI返回结果 ===");
    console.log(completion.choices[0].message.content);
  } catch (error) {
    console.error("API调用失败，请检查网络连接或API Key是否有误：", error.message);
  }
}

main();


**关键点解读：**

- **`dotenv.config()` 永远放在文件最顶部的第一件事**，确保在初始化 `OpenAI` 客户端之前，环境变量就已加载完毕
- 代码中**没有任何硬编码的字符串密钥**——所有敏感信息都通过 `process.env` 读取
- 有**简单的密钥缺失提示**：如果忘了配 `.env`，会直接报错，不会发送无效请求浪费额度
- `throw new Error` 可以帮你快速定位问题，避免空 key 在网络上失败导致混淆
