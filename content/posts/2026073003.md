## 核心步骤：从空项目到豆包接入

### 第一步：初始化Node.js项目并安装依赖

看好了，这一行都不能省：

bash
mkdir doubao-node-demo
cd doubao-node-demo
npm init -y
npm install openai dotenv


`openai` 是OpenAI官方Node.js库，[千聚ai大模型中转站](https://www.qianjuai.com/)接口完全兼容OpenAI格式，所以直接用。`dotenv` 是用来管理环境变量的，建议养成好习惯。

注意：别用 `http-proxy-middleware` 或者 `agent`，除非你确信自己需要。小白经常因为装了这玩意儿反而被代理规则坑死。

### 第二步：配置环境变量

新建一个 `.env` 文件，把API密钥和base url写进去：

env
QIANJU_AI_BASE_URL=https://www.qianjuai.com/v1
QIANJU_AI_API_KEY=sk-你的密钥


密钥严格保密，别提交到Git仓库。如果你不小心提交了，马上去千聚后台把这个密钥删了、重新生成一个。

### 第三步：写核心鉴权代码

新建 `index.js`，写入：

javascript
import { OpenAI } from 'openai';
import 'dotenv/config';

const client = new OpenAI({
  apiKey: process.env.QIANJU_AI_API_KEY,
  baseURL: process.env.QIANJU_AI_BASE_URL,
});

async function chatWithDoubao() {
  try {
    const response = await client.chat.completions.create({
      model: 'doubao-pro-32k',
      messages: [
        { role: 'user', content: '你好，用一句话介绍[千聚ai大模型中转站](https://www.qianjuai.com/)。' },
      ],
    });
    console.log('豆包回复：', response.choices[0].message.content);
  } catch (error) {
    console.error('接入出错了：', error);
  }
}

chatWithDoubao();


这里要注意一个坑：**模型名字必须用千聚平台定义的模型ID**。豆包大模型在[千聚ai大模型中转站](https://www.qianjuai.com/)里的对应ID是 `doubao-pro-32k`，别写成 `doubao` 或 `bytedance-doubao`（这些都不对）。

### 第四步：跑起来验证

终端运行：

bash
node index.js


如果你看到控制台输出“豆包回复：xxx”且没有报错，恭喜你——跑通了。

## 常见坑位，一个都别踩

### 1. Base URL 写错了或忘了加 `/v1`

这是出镜率最高的错误。[千聚ai大模型中转站](https://www.qianjuai.com/)的API接口是 `https://www.qianjuai.com/v1`，不是 `https://www.qianjuai.com`。少了 `/v1`，整个请求会走错路由，报 404 或 401。

### 2. model 名字不对

你如果是调用 `doubao-pro-32k`，就老老实实用这个ID。千聚支持豆包系列多个模型，具体有哪些——去官网文档查“可用模型列表”。别自己脑补名称。

### 3. 密钥权限问题

千聚后台生成的Key，默认是有权限限制的。你先确认这个Key有没有开通调用豆包模型的权限。在“API管理” -> “密钥详情”里，把目标模型勾上。

### 4. 没有用最新的 Node.js

如果你用 v16，`openai` 包可能需要额外适配。强烈建议直接上 v18 以上。如果你真的必须用 v16，那装 `openai@3` 版本，但接口调用方式会不一样。

## 验证成功的标志：不只这一次

单次调用成功不等于你完全学会了。建议在接入后，再跑两个测试：

### 测试1：流式输出

javascript
async function streamChat() {
  const stream = await client.chat.completions.create({
    model: 'doubao-pro-32k',
    messages: [{ role: 'user', content: '讲一个三分钟的故事' }],
    stream: true,
  });
  for await (const chunk of stream) {
    process.stdout.write(chunk.choices[0]?.delta?.content || '');
  }
}
streamChat();


### 测试2：函数调用

javascript
const response = await client.chat.completions.create({

    type: 'function',
