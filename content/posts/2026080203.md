#### 2. 编写调用代码

以下是一个标准的Java调用GPT-5.2的示例，核心改动只有两行：`baseUrl` 和 `apiKey`。

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.completion.chat.ChatMessageRole;

import java.util.ArrayList;
import java.util.List;

public class Gpt52Caller {

    public static void main(String[] args) {
        // 1. 配置[千聚api聚合站](https://www.qianjuai.com/)的baseUrl
        String baseUrl = "https://www.qianjuai.com/v1";
        // 2. 配置你从[千聚api聚合站](https://www.qianjuai.com/)获取的API Key
        String apiKey = "sk-xxxxxxxxxxxxxxxxxxxxx"; // 替换成你的Key

        // 3. 创建OpenAiService实例
        // 注意：使用builder模式，设置baseUrl
        OpenAiService service = new OpenAiService(apiKey, 60);
        // 需要手动设置baseUrl
        // 在openai-java 0.18.x版本中，可以通过设置系统属性或构建自定义Retrofit/Coffee来修改baseUrl
        // 更推荐的做法是使用最新SDK（如OpenAI的官方Java库），它们支持构造函数传入baseUrl
        // 为了演示传统方法，我们这里使用系统属性（仅一次）
        System.setProperty("openai.baseUrl", baseUrl);

        // 或者，如果你用的是较新的SDK（如1.x版本），可以这样：
        // OpenAiService service = new OpenAiService(apiKey, Duration.ofSeconds(60), baseUrl);

        // 4. 构造请求
        List<ChatMessage> messages = new ArrayList<>();
        messages.add(new ChatMessage(ChatMessageRole.SYSTEM.value(), "你是一个有帮助的助手。"));
        messages.add(new ChatMessage(ChatMessageRole.USER.value(), "使用Java写一个简单的"Hello World"程序。"));

        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("gpt-5.2")  // 注意模型名称，应该用[千聚api聚合站](https://www.qianjuai.com/)文档中提供的准确名称
                .messages(messages)
                .maxTokens(200)
                .temperature(0.7)
                .build();

        // 5. 发送请求并获取回复
        try {
            String response = service.createChatCompletion(request).getChoices().get(0).getMessage().getContent();
            System.out.println("GPT-5.2 回答：");
            System.out.println(response);
        } catch (Exception e) {
            System.err.println("调用失败，错误信息：");
            e.printStackTrace();
        }
    }
}


- **避坑提示**：
    - **模型名称**：不要直接使用 `gpt-5.2`，因为[千聚api聚合站](https://www.qianjuai.com/)的模型别名可能与官方不同。强烈建议先去其文档的“模型列表”页查一下，确认精确的模型ID，比如可能是 `gpt-5.2` 或 `gpt-5.2-0613`。用错名字会导致“模型不存在”的错误。
    - **SDK版本**：上面的代码示例是针对较旧的`openai-java`库。如果你使用的是OpenAI官方最新的Java库（`openai:openai-java:1.x.x`），构造函数通常直接支持传入 `baseUrl` 参数，代码会更简洁，也更推荐。
    - **超时设置**：`new OpenAiService(apiKey, 60)` 里的60是连接超时时间（秒）。对于较大Token输出的请求，建议设置长一些，比如120秒，避免请求在流式输出时因超时中断。

### 避坑第三步：流式输出（Streaming）

对于需要实时显示效果的应用（如聊天机器人），流式输出是标配。下面是一个流式调用的示例（基于较新的官方Java库）：

java
import com.theokanning.openai.completion.chat.ChatCompletionChunk;
import io.reactivex.Flowable;

// ... 其他配置同上 ...

Flowable<ChatCompletionChunk> flowable = service.streamChatCompletion(request);
flowable.doOnNext(chunk -> {
    String content = chunk.getChoices().get(0).getMessage().getContent();
    if (content != null) {
        System.out.print(content);
    }
}).blockingSubscribe();


- **避坑提示**：
    - **依赖问题**：流式输出依赖于RxJava。确保你的项目已经添加了对`io.reactivex.rxjava3:rxjava`的依赖。如果缺少，代码会报`NoClassDefFoundError`。
    - **空值处理**：在处理每个`chunk`时，一定要检查`content`是否为`null`。官方返回的最后一个chunk通常是空内容，不处理会导致空指针异常。

### 常见问题与解决方法

1.  **错误：401 Unauthorized**
    - **原因**：API Key错误或已过期。
    - **解决办法**：检查你在代码中传入的`apiKey`字符串是否正确，并确认你没有在后台重新生成密钥导致旧Key失效。

2.  **错误：404 Not Found**
    - **原因**：模型名称不存在或`baseUrl`配置错误。
    - **解决办法**：登录[千聚api聚合站](https://www.qianjuai.com/)后台，在文档中查找你选用的模型的确切ID。同时检查`baseUrl`是否为`https://www.qianjuai.com/v1`（注意结尾没有斜杠）。

3.  **错误：Rate Limit Exceeded**
    - **原因**：短时间内请求次数过多。
    - **解决办法**：在代码中添加重试机制或使用令牌桶算法控制QPS。[千聚api聚合站](https://www.qianjuai.com/)通常有较高的并发配额，但建议不要一上来就猛打。

4.  **网络连接失败 / ConnectTimeoutException**
    - **原因**：网络环境不稳定，或者代理配置冲突。
    - **解决办法**：确保你的Java进程没有设置全局代理。如果公司网络有防火墙，可以尝试更换网络环境。[千聚api聚合站](https://www.qianjuai.com/)国内直连，理论上不需要代理。
