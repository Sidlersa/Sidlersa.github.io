## 另一种频率限制：Rate Limit 如何避坑

接口不报 502 了，但突然返回 429（Too Many Requests），或者干脆无响应。这说明你触发了平台的限流。

[千聚ai聚合平台](https://www.qianjuai.com/)的免费额度可以让你测试，但生产环境需要控制并发：

- 单个 API key 每秒不超过 10 次请求（标准推荐，但平台无硬性限制，超量会降速）。
- 遇到 429 时，不要立即重试，等 1-3 秒。
- 使用“指数退避”重试策略：第一次失败等 1 秒，第二次 2 秒，第三次 4 秒，以此类推。代码示例：

javascript
const sleep = (ms) => new Promise(resolve => setTimeout(resolve, ms));
async function retryableCall() {
  for (let i = 0; i < 3; i++) {
    try {
      return await callClaudeHaiku();
    } catch (e) {
      if (e.response?.status === 429 || e.code === 'ETIMEDOUT') {
        const waitTime = Math.pow(2, i) * 1000; // 2^i 秒
        console.log(`第${i+1}次失败，等待${waitTime}ms后重试`);
        await sleep(waitTime);
      } else {
        throw e;
      }
    }
  }
  console.error('重试3次均失败');
}


这样你的代码在面对限流时就不会崩溃，而是优雅重试。
