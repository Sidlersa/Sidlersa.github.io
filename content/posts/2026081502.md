#### 步骤 1：环境准备和安装

#### 步骤 2：创建服务端调用代码

我们写一个函数，用来判断用户留言的情绪是“满意”、“不满意”还是“模糊”。

javascript
import OpenAI from 'openai';

// 关键点：配置[千聚ai大模型中转站](https://www.qianjuai.com/)的服务地址
const client = new OpenAI({
    apiKey: process.env.QIANJU_API_KEY, // 在千聚平台申请的key
    baseURL: 'https://www.qianjuai.com/v1',
});

async function analyzeSentiment(userText) {
    const response = await client.chat.completions.create({
        // 在这里指定模型为 GPT-5nano
        model: 'gpt-5nano',
        messages: [
            {
                role: 'system',
                content: '请判断以下用户反馈的情绪。仅输出三个选项中的一个："满意"，"不满意"，或者"模糊"。不要输出任何其他文字或标点。',
            },
            { role: 'user', content: userText },
        ],
        // 核心优化点：控制输出的 token 数量
        max_tokens: 10,
        temperature: 0.1,
    });

    return response.choices[0].message.content.trim();
}

// 测试调用
const testFeedback1 = '你们的应用终于不卡了，太棒了！';
const testFeedback2 = '什么鬼产品，一点都不好用。';

console.log('测试1:', await analyzeSentiment(testFeedback1)); // 预期输出: 满意
console.log('测试2:', await analyzeSentiment(testFeedback2)); // 预期输出: 不满意

// 这样一次调用成本就在 0.003 元左右！


#### 步骤 3：进一步压榨性能

1.  **控制输出长度**：用 `max_tokens` 限制模型不要“发散”，既然只需要三个字的结果，10 个 token 足够了。
2.  **降低 Temperature**：设为 0.1 让输出更确定，减少重复和不必要的尝试。
3.  **精简 Prompt**：不要写废话，指令越精确，Token消耗越少。刚才的例子如果加一堆修辞，成本就上去了。
