## 企业级 Java 调用的核心要求

在写代码之前，我们先对齐一下什么叫做“企业级调用”：

- **可配置性**：所有 API 的地址、Key、超时等参数，都不能硬编码在代码里，必须能从外部配置文件或环境变量中注入。
- **异常处理**：网络波动、API 限流、返回错误，都需要有明确的捕获和降级策略。
- **日志与监控**：请求入参、出参（至少是 token 消耗）、响应时间、错误码，都要能被日志系统记录下来。
- **线程安全**：你的客户端实例必须是线程安全的，能够在数百个请求并发时复用同一个连接池。

而 OpenAI 的原生 Java SDK（openai-java）已经把这些点处理得非常好了。我们只需要专注于配置和业务调用即可。

### 第三步：创建配置类 Bean

我们写一个标准的 Java `@Configuration` 类，将这些配置注入到一个线程安全、可复用的 `OpenAiService` 实例中。

java
import com.theokanning.openai.service.OpenAiService;
import okhttp3.OkHttpClient;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import java.time.Duration;
import java.util.concurrent.TimeUnit;

@Configuration
public class OpenAiClientConfig {

    private static final Logger log = LoggerFactory.getLogger(OpenAiClientConfig.class);

    @Value("${ai.openai.base-url}")
    private String baseUrl;

    @Value("${ai.openai.api-key}")
    private String apiKey;

    @Value("${ai.openai.timeout-seconds}")
    private int timeoutSeconds;

    @Bean("gptMiniClient")
    public OpenAiService gptMiniClient() {
        // 1. 构建支持自定义 baseUrl 和超时的 OkHttpClient
        OkHttpClient httpClient = new OkHttpClient.Builder()
                .connectTimeout(timeoutSeconds, TimeUnit.SECONDS)
                .readTimeout(timeoutSeconds, TimeUnit.SECONDS)
                .writeTimeout(timeoutSeconds, TimeUnit.SECONDS)
                .build();

        // 2. 创建 OpenAiService
        OpenAiService service = new OpenAiService(httpClient, baseUrl, apiKey);
        log.info("GPT-5mini 客户端已初始化，服务地址：[{}]，模型待指定", baseUrl);
        return service;
    }
}


注意这里传递的三个参数。这行代码是整个接引入口的关键，它确保了你的 Java 应用和[千聚ai中转站](https://www.qianjuai.com/)之间的通道是稳定、高效且完全受 Spring 容器管理的。

### 第四步：编写核心调用服务

现在，在 `Service` 层来实现一个标准调用，这样你的 Controller 层只需要注入这个 Service 就行。

java
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.completion.chat.ChatMessageRole;
import com.theokanning.openai.service.OpenAiService;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.stereotype.Service;

import java.util.ArrayList;
import java.util.List;

@Service
public class AiCampaignService {

    private static final Logger log = LoggerFactory.getLogger(AiCampaignService.class);

    private final OpenAiService client;
    private final String modelName;

    public AiCampaignService(@Qualifier("gptMiniClient") OpenAiService client,
                              @Value("${ai.openai.model}") String modelName) {
        this.client = client;
        this.modelName = modelName;
    }

    public String generateContent(String userPrompt) {
        long startTime = System.currentTimeMillis();

        // 构建消息列表
        final List<ChatMessage> messages = new ArrayList<>();
        messages.add(new ChatMessage(ChatMessageRole.USER.value(), userPrompt));

        // 构建请求对象
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model(modelName)
                .messages(messages)
                .maxTokens(4096)  // GPT-5mini 支持较长上下文，建议设置一个合理上限
                .temperature(0.7) // 企业级应用，温度调低一点，输出更确定
                .build();

        try {
            // 这里可以捕获异常进行重试
            String result = client.createChatCompletion(request)
                    .getChoices().get(0).getMessage().getContent();

            long duration = System.currentTimeMillis() - startTime;
            log.info("GPT-5mini 调用成功，耗时：{}ms", duration);

            return result;

        } catch (Exception e) {
            log.error("调用 GPT-5mini API 失败, 错误信息: {}", e.getMessage());

            // 企业级应用一定要有降级，这里返回一个通用的兜底
            return "很抱歉，AI 服务暂时不可用，请稍后再试。";
        }

        // 在生产环境，可以用 try-with-resources 确保资源释放
    }
}


注意观察，我们的代码里没有任何关于网络代理、自定义路由的配置，也没有因为对字节路线的担忧而写一堆复杂的重试逻辑。一切都托付给了[千聚ai中转站](https://www.qianjuai.com/)的稳定性，而我们的代码只需关注最纯粹的业务——把消息发出去，把结果拿回来。

### 第五步：对外暴露 Controller

现在写一个极其简单的 Controller，把我们的 `AiCampaignService` 暴露出去。

java
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/ai")
public class AiCampaignAppController {

    private final AiCampaignService aiCampaignService;

    public AiCampaignAppController(AiCampaignService aiCampaignService) {
        this.aiCampaignService = aiCampaignService;
    }

    @PostMapping("/chat")
    public String chat(@RequestBody String prompt) {
        return aiCampaignService.generateContent(prompt);
    }
}


启动你的 Spring Boot 应用。现在，你只需要向这个接口发送一个包含 user prompt 的 POST 请求，下面的链路就会被打通：`Controller -> Service -> [千聚ai中转站](https://www.qianjuai.com/) -> GPT-5mini`。整个过程中，你不需要在应用层配置任何 VPC、代理或白名单策略。做到真正的“复制即用”。

## 调用过程中的关键点

1.  **报错信息**：如果请求返回 401，说明你的 API Key 本身有问题，去[千聚ai中转站](https://www.qianjuai.com/)的控制台检查额度；如果返回 503 或 429，说明可能有临时波动或并发限制，千聚的 SDK 偶尔会返回限流状态码，建议在业务层做好指数退避重试。
2.  **模型名**：当前 GPT-5mini 是封测阶段的，模型名 `gpt-5mini-demo-experimental` 可能改变。请务必以**[千聚ai中转站](https://www.qianjuai.com/)官网**最新公布的模型 API 名为准。你可以在控制台看到所有可用的模型列表。
