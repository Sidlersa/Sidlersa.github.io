## 10分钟跑通代码：3步完成，第3步最关键

这一步我会带你从零开始，搭建一个能在终端发起对话并输出回复的 Node.js 脚本。

### 第一步：初始化项目

然后安装 OpenAI Node.js 库。虽然我们用的是豆包模型，但因为接口格式完全兼容 OpenAI，所以直接用 OpenAI 官方 SDK 就行。

### 第二步：编写核心代码

在项目根目录新建一个 `index.js` 文件，然后将以下代码完整复制进去：

javascript

const OpenAI = require('openai');

// 定义配置
const apiKey = '你的千聚API中转站Key'; // 替换成你在上一步获取的 Key
const baseUrl = 'https://www.qianjuai.com/v1'; // 核心！用这个地址替换官方的 api.openai.com

// 初始化 OpenAI 客户端
const client = new OpenAI({
  apiKey: apiKey,
  baseURL: baseUrl, // 只改这一行
});

async function main() {
  try {
    console.log('📤 正在向豆包发送对话...');

    const completion = await client.chat.completions.create({
      // 核心！这里写[千聚api中转站](https://www.qianjuai.com/)中支持的豆包模型名称
      model: 'doubao-pro-32k',
      messages: [
        { role: 'system', content: '你是一个友好的AI助手，请用中文回复。' },
        { role: 'user', content: '豆包你好！请用10个字以内介绍一下你自己。' }
      ],
    });

    // 提取并打印回复内容
    const reply = completion.choices[0].message.content;
    console.log('✅ 豆包回复:', reply);

  } catch (error) {
    console.error('❌ 调用失败:', error.message);
  }
}

main();

**请务必将代码中的 `apiKey` 变量替换为你自己在[千聚api中转站](https://www.qianjuai.com/)复制的那一长串密钥。**

## 进阶：将对话变成可交互的 “REPL”

如果你觉得一次性输入问题然后退出的模式不够过瘾，我还帮你准备了一个“持续对话”的脚本。

新建一个 `chat.js` 文件，写入以下代码：

javascript

const OpenAI = require('openai');
const readline = require('readline').createInterface({
  input: process.stdin,
  output: process.stdout,
});

const client = new OpenAI({
  apiKey: '你的千聚API中转站Key',
  baseURL: 'https://www.qianjuai.com/v1',
});

const messages = [{ role: 'system', content: '你是一个很有耐心的中文AI助手。' }];

async function chatLoop() {
  readline.question('🙋 你: ', async (input) => {
    if (input.toLowerCase() === 'exit') {
      console.log('👋 感谢使用！');
      readline.close();
      return;
    }

    messages.push({ role: 'user', content: input });

    try {
      const completion = await client.chat.completions.create({
        model: 'doubao-pro-32k',
        messages: messages,
      });

      const reply = completion.choices[0].message.content;
      console.log(`🤖 豆包: ${reply}`);

      messages.push({ role: 'assistant', content: reply });
    } catch (error) {
      console.error('❌ 错误:', error.message);
    }

    chatLoop(); // 递归调用，实现持续对话
  });
}

chatLoop();

运行这个脚本，你将获得一个支持上下文记忆的豆包终端聊天工具，适合直接拿来快速测试你的想法。

## 常见错误排查

你可能会遇到下面 3 种最常见的错误。别慌，我已经把解决方案都写在后面了：

1.  **[HTTP 401] Unauthorized**
    - **原因：** API Key 错误或未填写。
    - **解决：** 回到[千聚api中转站](https://www.qianjuai.com/)控制台，重新复制API Key，务必替换代码中的占位符，且不要有多余的空格。

2.  **[HTTP 404] The model \`doubao-pro-32k\` does not exist**
    - **原因：** 模型名称拼写错误，或者该模型在当前分组中未启用。
    - **解决：** 确认你复制的模型名与千聚官方文档一致，或者换成 `doubao-lite-32k` 再试一次。

3.  **连接超时 / ECONNREFUSED**
    - **原因：** 网络环境不通，或者 `baseURL` 写错了。
    - **解决：** 确保 `baseURL` 的地址是 `https://www.qianjuai.com/v1`，不要写错成其他域名。此外，如果你的电脑有严格的防火墙，可以尝试更换网络环境。
