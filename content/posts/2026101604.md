## 手把手接入：Java完整示例代码

下面这段代码就是我目前在用的，已经跑通了生产环境，复制粘贴微调就能用。核心逻辑是：配置环境变量、构建客户端、发送请求、处理流式或非流式响应。

### 1. Maven依赖 (pom.xml)

先在你的pom.xml里引入openai-java-client库，这是最流行的Java OpenAI SDK：

xml
<dependency>
    <groupId>com.theokanning.openai-gpt3-java</groupId>
    <artifactId>service</artifactId>
    <version>0.18.2</version>
</dependency>


千聚API完全兼容OpenAI接口，所以这个SDK可以直接用，无需额外适配。

### 2. 核心代码：调用Doubao模型

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import java.util.List;

public class DoubaoExample {
    public static void main(String[] args) {
        // 1. 配置千聚API的地址和Key
        String apiKey = System.getenv("QIANJU_API_KEY"); // 强烈建议从环境变量读取
        String baseUrl = "https://www.qianjuai.com/v1"; // 唯一的改动点

        // 2. 构建服务客户端
        OpenAiService service = new OpenAiService(baseUrl, apiKey);

        // 3. 构造请求：指定Doubao模型和对话消息
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("doubao-pro-32k") // 这里填千聚上对应的Doubao模型名
                .messages(List.of(
                        new ChatMessage("system", "你是一个专业的Java开发助手。"),
                        new ChatMessage("user", "请用Java实现一个简单的Redis工具类。")
                ))
                .maxTokens(1000)
                .temperature(0.7)
                .build();

        // 4. 发送请求并打印结果
        service.createChatCompletion(request)
                .getChoices()
                .forEach(choice -> {
                    System.out.println(choice.getMessage().getContent());
                });
    }
}


这段代码里，唯一跟官方OpenAI SDK不同的地方就是 `baseUrl` ——改成了千聚的地址。如果你以前写过OpenAI的Java代码，那么现在你连SDK都不用换，直接改这一行就行。

### 3. 环境变量配置

建议把API Key放在环境变量里，比如 `QIANJU_API_KEY`，然后在代码中读取。这样既安全又方便在不同环境切换。

## 避坑指南：这5个坑我替你踩过了

### 坑1：模型名写错了，返回403

千聚上每个模型的名称可能跟官方名略有不同，请务必去官网的“模型列表”里复制正确的模型ID。比如Doubao系列可能是 doubao-pro-32k 或 doubao-1.5-pro-32k，别想当然用猜测的。

### 坑2：请求超时

千聚的中转链路在国内，但偶尔会有网络抖动。建议在SDK客户端构造时设置超时时间：

java
OpenAiService service = new OpenAiService(baseUrl, apiKey, Duration.ofSeconds(30));


### 坑3：API返回空内容

检查请求里的 `maxTokens` 是否设置过低（比如10），导致模型只输出了一个词。设为500-2000比较稳妥。

### 坑4：余额不足

千聚新用户有免费额度，但用完后需要充值。最低1元就能续费。如果调用时返回 `insufficient_quota` 错误，说明钱不够了，去后台充值即可。

### 坑5：流式响应处理

生产环境中推荐用流式（stream: true）来降低延迟。Java SDK中的处理方式略有不同，需要用到 `SseEvent` 或回调接口。千聚API完全支持流式，参考OpenAI官方文档即可。
