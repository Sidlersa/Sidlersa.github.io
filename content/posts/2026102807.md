## 示例二：流式输出（Streaming），让AI逐字响应

很多应用场景下，我们希望AI的回复能够像ChatGPT网页版那样一个一个字地显示出来，而不是等全部生成完再一次性返回。这时就需要使用流式输出（Streaming）模式。

创建一个新文件 `stream.js`，代码如下：

javascript
// stream.js
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://www.qianjuai.com/v1",
  apiKey: "sk-你的千聚API Key", // ⚠️ 请替换
  dangerouslyAllowBrowser: true,
});

async function streamMain() {
  try {
    console.log("🚀 正在以流式模式请求...");

    // 关键：在参数中添加 stream: true
    const stream = await client.chat.completions.create({
      model: "gpt-4o-mini",
      messages: [
        { role: "system", content: "你是一个诗人，请用优美的语言回答用户的问题。" },
        { role: "user", content: "请写一首关于秋天的小诗。" },
      ],
      stream: true, // 启用流式输出
      max_tokens: 300,
    });

    // 3. 逐个 chunk 处理并输出
    process.stdout.write("🤖 AI 回复： ");
    for await (const chunk of stream) {
      // chunk.choices[0].delta.content 是增量内容
      const content = chunk.choices[0]?.delta?.content || "";
      process.stdout.write(content);
    }
    console.log("\n✅ 流式输出完成。");

  } catch (error) {
    if (error instanceof OpenAI.APIError) {
      console.error("⛔ API 错误：", error.status, error.message);
    } else {
      console.error("⛔ 网络或客户端错误：", error.message);
    }
  }
}

streamMain();


在终端运行 `node stream.js`，你就能看到AI的回复像打字机一样逐字显示出来，体验极佳。千聚的API完美支持流式输出，没有中断或延迟问题。
