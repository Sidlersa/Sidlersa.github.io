## 第一种方式：使用OpenAI Java SDK调用

这是最推荐的方式——如果你之前写过OpenAI的Java调用，那代码几乎不用改。

java
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.service.OpenAiService;
import java.time.Duration;
import java.util.Arrays;

public class DeepSeekCaller {
    public static void main(String[] args) {
        // 创建服务实例，千聚的BASE_URL + 你的Key
        OpenAiService service = new OpenAiService(
            DeepSeekConfig.API_KEY,
            Duration.ofSeconds(60)
        );
        service.setBaseUrl(DeepSeekConfig.BASE_URL);

        // 构建消息列表
        ChatMessage systemMessage = new ChatMessage("system", "你是一个资深的Java程序员，擅长代码生成和解释。");
        ChatMessage userMessage = new ChatMessage("user", "请用Java写一个冒泡排序算法，并生成注释。");

        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model(DeepSeekConfig.MODEL)
                .messages(Arrays.asList(systemMessage, userMessage))
                .temperature(0.7)
                .maxTokens(2048)
                .build();

        // 发送请求并获取回复
        service.createChatCompletion(request)
                .getChoices()
                .forEach(choice -> System.out.println(choice.getMessage().getContent()));
        
        // 记得关闭服务释放资源
        service.shutdownExecutor();
    }
}


跑一下main方法，控制台里应该就能看到DeepSeek Coder生成的冒泡排序代码，带注释。如果出错了，99%的情况是你把`base_url`写成了`https://www.qianjuai.com`而忘记加`/v1`——别笑，我犯过这个错误。

## 第二种方式：流式调用（实时输出）

如果要在界面上或者命令行里看到打字机效果，需要改成流式调用。代码也非常简单：

java
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.service.OpenAiService;
import io.reactivex.Flowable;
import java.time.Duration;
import java.util.Arrays;

public class DeepSeekStreamCaller {
    public static void main(String[] args) {
        OpenAiService service = new OpenAiService(
            DeepSeekConfig.API_KEY,
            Duration.ofSeconds(120) // 流式调用设置更长的超时时间
        );
        service.setBaseUrl(DeepSeekConfig.BASE_URL);

        ChatMessage userMessage = new ChatMessage("user", "用Java写一个线程安全的单例模式，要求懒加载。");

        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model(DeepSeekConfig.MODEL)
                .messages(Arrays.asList(userMessage))
                .temperature(0.3)
                .maxTokens(4096)
                .stream(true) // 开启流式
                .build();

        // 流式接收数据，逐块打印
        Flowable<ChatCompletionChunk> flowable = service.streamChatCompletion(request);
        flowable.blockingForEach(chunk -> {
            String content = chunk.getChoices().get(0).getMessage().getContent();
            if (content != null) {
                System.out.print(content);
            }
        });

        service.shutdownExecutor();
    }
}


跑起来之后，你会看到控制台里文字一行一行地蹦出来——这就是流式输出的效果。千聚的链路在国内直连下非常稳定，流式输出的延迟基本上和调用官方API持平，没有额外的缓冲延迟。

## 第三种方式：纯HttpClient调用（不依赖第三方SDK）

如果你所在的项目对依赖管理极其严格，或者你只是想要最轻量的调用方式，直接用`java.net.http.HttpClient`也能搞定。

java
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import java.time.Duration;

public class DeepSeekHttpClient {
    public static void main(String[] args) throws Exception {
        HttpClient client = HttpClient.newBuilder()
                .connectTimeout(Duration.ofSeconds(30))
                .build();

        String requestBody = """
                {
                    "model": "deepseek-coder",
                    "messages": [
                        {"role": "user", "content": "用Java写一个快速排序"}
                    ],
                    "temperature": 0.5,
                    "max_tokens": 2048
                }
                """;

        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create("https://www.qianjuai.com/v1/chat/completions"))
                .header("Content-Type", "application/json")
                .header("Authorization", "Bearer " + DeepSeekConfig.API_KEY)
                .POST(HttpRequest.BodyPublishers.ofString(requestBody))
                .build();

        HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
        System.out.println(response.body());
    }
}


这种方式的好处是什么都不需要安装，但你需要自己解析JSON响应。如果你已经对Gson或Jackson很熟悉了，这种方式也挺干净的。

## 常见报错及解决方法

写到这里，我不妨把最常遇到的几个问题列出来，你遇到了可以直接抄解决方案：

**1. 报错“401 Unauthorized”**
检查你的API Key是不是完整粘贴进去了。有时候复制时会把空格或者换行符带进去。另外确认Key是在千聚平台创建的，不是OpenAI的。

**2. 报错“404 Not Found”**
检查base_url是否写成了`https://www.qianjuai.com`（缺少`/v1`）。正确的地址是：`https://www.qianjuai.com/v1`。

**3. 报错“Connection refused”或者请求超时**
这个一般是你所在的网络环境需要代理才能访问外网。千聚是支持国内直连的，如果你还报这个错误，看看是不是你的IDE或者JVM设置里强制走了代理。检查一下`HTTP_PROXY`和`HTTPS_PROXY`环境变量。

**4. 返回空内容或者错误说模型不存在**
确认你填写的模型名称是`deepseek-coder`。千聚支持的模型名称可能有细微差异，最好在平台的控制台里查看最新的模型ID列表。

**5. 流式输出卡住不继续**
这种情况通常是因为JVM的DNS解析不稳定。可以在启动参数里加`-Djava.net.preferIPv4Stack=true`，强制使用IPv4。另一个方案是检查你的`HttpClient`超时设置，把`Duration.ofSeconds`值设大一点（比如120秒）。
