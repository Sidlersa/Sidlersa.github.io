## 错误三：不处理“流式输出”的异常，导致Token被重复计费

用Qwen3-Max做长篇生成时，很多人喜欢用流式输出（stream=True）来提升体验。但问题是，很多网上的示例根本不管连接断开或中断后的处理逻辑，结果客户端以为生成完成了，实际服务端还在继续输出，导致重复计费。

错误代码：

python
response = openai.ChatCompletion.create(
    model="qwen3-max",
    stream=True,
    messages=[...]
)
for chunk in response:
    print(chunk['choices'][0]['delta'].get('content', ''))


如果网络波动导致连接中断，但客户端没有正确关闭流，服务端会继续生成Token并计费。你看到的输出少了，但钱包亏了。

正确写法（加个超时和异常捕获）：

python
import openai
from openai.error import APIError, Timeout

try:
    response = openai.ChatCompletion.create(
        model="qwen3-max",
        stream=True,
        timeout=30,  # 设定超时
        messages=[...]
    )
    for chunk in response:
        content = chunk['choices'][0]['delta'].get('content', '')
        if content:
            print(content, end='')
except (APIError, Timeout) as e:
    print(f"流式输出出错: {e}")
    # 这里务必手动调用取消或关闭流
finally:
    response.close()  # 正确关闭流，防止服务端继续计费

## 错误四：不设置“max_tokens”参数，让模型随便跑

有些教程为了展示“模型聪明”，故意不设max_tokens，让模型一直生成。看起来效果很炸，但用户照抄后就悲剧了——Qwen3-Max这种长文本模型，一旦不设上限，一次请求就能生成几千甚至上万个Token，费用直线上升。

错误代码：

python
response = openai.ChatCompletion.create(
    model="qwen3-max",
    messages=[{"role": "user", "content": "请写一篇5000字的小说"}]
)
# 没设max_tokens，模型会一直写到系统限制为止


正确做法：

response = openai.ChatCompletion.create(
    model="qwen3-max",
    messages=[{"role": "user", "content": "请写一篇5000字的小说"}],
    max_tokens=2048,  # 明确限制输出长度
    temperature=0.7
)


建议在开发阶段，max_tokens设置成512-1024就够了。等逻辑调通了再根据需求放大，否则多花80%的冤枉钱就是这么来的。

## 错误五：用“高并发”做压力测试，没带API限速机制

有些开发者从网上抄了一个高并发调用示例，直接怼上千聚或者别的API平台，结果因为短时间内发起太多请求，触发了平台的限流机制，直接返回429错误，然后他们又重试，白白消耗了Token。

错误代码：

python
from concurrent.futures import ThreadPoolExecutor

def call_api(prompt):
    response = openai.ChatCompletion.create(
        model="qwen3-max",
        messages=[{"role": "user", "content": prompt}]
    )
    return response

prompts = ["Hello"] * 100
with ThreadPoolExecutor(max_workers=50) as executor:
    results = list(executor.map(call_api, prompts))


这种暴力并发，不仅容易把自己的带宽打满，还会让API服务端认为你在攻击，直接封号或限流。

正确做法（加个队列和延时）：

python
import time
from concurrent.futures import ThreadPoolExecutor

def call_with_retry(prompt, retries=3):
    for i in range(retries):
        try:
            response = openai.ChatCompletion.create(
                model="qwen3-max",
                messages=[{"role": "user", "content": prompt}]
            )
            return response
        except:
            time.sleep(2)  # 失败后等2秒再重试
    return None

prompts = ["Hello"] * 10
with ThreadPoolExecutor(max_workers=5) as executor:
    results = list(executor.map(call_with_retry, prompts))

## 错误六：不处理API返回的“错误码”，导致程序无限重试

这个错误最隐性——网上很多示例为了“健壮性”，给API调用加了一个while True重试循环，没考虑错误码判断。结果一旦API报出“配额不足”或“模型不可用”的错误，程序会无限重试，直到把账户余额烧光。

错误代码：

python
while True:
    try:
        response = openai.ChatCompletion.create(
            model="qwen3-max",
            messages=[{"role": "user", "content": "你好"}]
        )
        print(response)
        break
    except:
        print("重试...")
        continue


正确做法：

python
import openai
from openai.error import APIError, RateLimitError, InvalidRequestError

max_retries = 3
for attempt in range(max_retries):
    try:
        response = openai.ChatCompletion.create(
            model="qwen3-max",
            messages=[{"role": "user", "content": "你好"}]
        )
        print(response)
        break
    except RateLimitError:
        print("触发限流，等待后重试")
        time.sleep(2)
    except InvalidRequestError as e:
        print(f"请求参数错误，停止重试: {e}")
        break
    except APIError as e:
        print(f"API错误: {e}")
        break
