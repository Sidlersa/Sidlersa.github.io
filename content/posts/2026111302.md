## 常见错误与排查

很多人会卡在这里，我遇到过两个高频问题：

**1. 超时错误：`ETIMEDOUT` 或 `ECONNRESET`**
- 原因：你的网络环境DNS解析不稳定，或者CDN缓存问题
- 解决办法：在代码顶部设置`process.env.HTTP_PROXY`为空，或者确保你的Node版本支持TLS 1.3。

**2. `401 Unauthorized` 错误**
- 原因：API key格式不正确或已过期
- 解决办法：检查千聚ai后台的key是否有效，注意不要有额外空格。

**3. 模型名称错误：`model not found`**
- 原因：拼写错误或模型未上线
- 解决办法：确保模型名是`glm-4.5`，而不是`glm-4`或`GLM-4.5`（注意大小写）。千聚ai的模型列表里会标清楚每一个具体名称。

## 高级用法：流式输出与多轮对话

既然基础调用没问题了，我们再进阶一下。流式输出能让用户体验更好，这在聊天机器人中非常实用。

javascript
const stream = await client.chat.completions.create({
    model: 'glm-4.5',
    messages: [
        { role: 'user', content: '用markdown格式写一篇关于深度学习的文章' }
    ],
    stream: true,
});
for await (const chunk of stream) {
    process.stdout.write(chunk.choices[0]?.delta?.content || '');
}


注意：GLM-4.5支持多轮对话，你只需要在`messages`数组中依次传入`user`和`assistant`的对话历史即可。
