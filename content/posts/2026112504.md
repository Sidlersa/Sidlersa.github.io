## 为什么你的 API 总是被限流？

OpenAI 和 Anthropic 的官方 API 账户默认都有严格的速率限制。比如 GPT-4o，免费体验账户的 RPM（每分钟请求数）可能只有 3~5 次，TPM（每分钟 Token 数）更是低得可怜。你只是稍微多写了几行代码，或者同时开了几个会话，请求就直接被拦下来了。

问题还不止这些。官方 API 对 IP 地址很敏感，如果你用的是共享 IP 或者代理 IP，很容易被误判为恶意流量，触发更严格的限流。另外，官方 API 的额度是预付费制，一旦你的账户余额不足或者信用评级下降，API 调用就会立刻中断，没有任何缓冲余地。

对开发者来说，这就像一个双重打击：既要应对限流策略，又要应付账户本身的不稳定因素。
